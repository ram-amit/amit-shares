TL;DR

Affiliates report daily DEP volatility after the 2-day lock window, blocking partner payments and campaign optimization (Natural Intelligence ~$1M-$1.5M/mo). Snowflake shows the lock column is frozen correctly today for mature entities (CM lock = L3 day-2 score, 12,579/12,580 match). The most likely explanation is partners seeing unlocked DEP or full re-exports mixing entities still in the 0-2 day lock window, not a broken lock implementation. Cannot prove historical lock mutation (no CM time travel). Need affiliate user IDs + export pipeline audit to close.

Issue context (from call)

  • Partners (Natural Intelligence + others) receive DEP via daily reports, up to 8x/day. Each report re-exports all month-to-date events.
  • Payment model: 80% fixed + 20% tied to DEP performance. Entities with DEP >= $18 are payable (cap $1,000).
  • Field used: PREDICTED_FIRST_ARR_2_DAYS_LOCK (DEP2 lock). Partners expect it to freeze after 48h.
  • Reported behavior: same user ID shows different DEP day-over-day after day 2 (e.g. DEP=2 on Jul 12, DEP=4 on Jul 13).
  • No DEP model version change since mid-June. Treated as technical, not intentional rollout.

Solution options

Tier 1 — Fix root cause

  • Audit Natural Intelligence export query: confirm predicted_first_arr_2_days_lock only
  • DS/BI daily QA: entities age>3 where lock ≠ L3 day-2 (should be 0%)

Tier 2 — Partner reporting hardening (parallel)

  • Partner snapshot table at T+2 (immutable contractual value)
  • Append-only reporting: once reported, never revise DEP
  • Export filter: only entities age ≥ 3 days

Tier 3 — Commercial (Aug touch base)

  • Review 80/20 split (test 70/30 or 90/10)
  • Market-vector DEP (different audiences, different signals)
  • Cohort-level payment vs per-user precision
